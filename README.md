# Muse-Spark-website-building-test
I tested the new Meta AI Muse-Spark by building a website. Here is what it built after about 5-6 prompts later. No code, no manual edits, just pure prompt coding.

Northstar Consulting — Muse Spark website build

I tested Meta's new **Muse Spark** model by building a complete marketing studio website from scratch. No code written by me, no manual edits — just pure prompt engineering.

**Result after 5–6 prompts:** a single-file, production-ready site with hash routing, GSAP transitions, Lenis smooth scroll, Three.js background, and a warm editorial design.

Live demo: https://o1ie12.github.io/Meta-AI-Muse-Spark-Website-Test/

## What I asked for

> "Build a complete single-file website for 'Northstar Consulting' — a thoughtful marketing and brand studio. Warm editorial aesthetic (ivory #FAF7F2, charcoal #1A1A1A, sage #8B9B8F, sand #E6DDD3, terracotta #B66A4E). Fraunces for headlines, Inter for body. 6 pages with hash routing, GSAP fade transitions, Lenis smooth scroll, subtle Three.js blobs, custom cursor."

Muse Spark delivered it in one shot, then iterated on two bugs (home image placement and modal scrolling).

## What it built

- **6 pages:** Home, Services, Work, Studio, Journal, Contact
- **Interactions:** smooth page transitions, mobile hamburger, work filters, expandable service cards, scrollable modals, form validation
- **Design details:** 16px radius, wool texture overlay, hover states, fully responsive
- **Tech:** 100% vanilla HTML/CSS/JS, no build step, ~45KB

All in a single `index.html` file.

## The Gemini test

I gave the exact same prompt to Google Gemini for a fair comparison.

**Gemini strengths:** sharper copywriting, better pricing structure, more detailed content strategy
**Muse Spark strengths:** superior visual polish, atmosphere (Lenis + Three.js), editorial warmth, motion design

Both are impressive. Muse Spark felt less "vibe-coded" — more like a designer built it.
